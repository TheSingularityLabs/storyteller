# Contributing to Storyteller

Thank you for your interest in contributing to Storyteller!

## What to Contribute

### ✅ Welcome Contributions

- **Bug fixes** - Report and fix issues
- **Documentation** - Improve guides and examples
- **Template improvements** - Better script formats
- **Validation rules** - Enhanced script checking
- **Examples** - More sample scripts
- **Layout patterns** - New visual composition ideas
- **Parser improvements** - Better extraction logic

### ❌ Not Appropriate

- API integrations (keep those in your own implementations)
- Proprietary prompt engineering
- Business-specific workflows
- API keys or credentials

## How to Contribute

1. **Fork the repository**
2. **Create a feature branch** (`git checkout -b feature/amazing-feature`)
3. **Make your changes**
4. **Add tests** if applicable
5. **Update documentation** as needed
6. **Commit your changes** (`git commit -m 'Add amazing feature'`)
7. **Push to the branch** (`git push origin feature/amazing-feature`)
8. **Open a Pull Request**

## Code Style

- Follow PEP 8 for Python code
- Use type hints where possible
- Add docstrings to functions
- Keep functions focused and small
- Write clear, descriptive commit messages

## Documentation

- Update relevant docs when adding features
- Include examples in docstrings
- Keep README.md up to date
- Add usage examples for new features

## Testing

- Test your changes with example scripts
- Ensure parser handles edge cases
- Verify layout selector works correctly

## Questions?

Open an issue for discussion before making large changes.

## Code of Conduct

Be respectful, inclusive, and constructive in all interactions.
